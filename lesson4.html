Lesson 4: The "Pre" and "Head" Tags

<Pre>

Let's say you want to put a poem into your webpage. You might copy and paste an existing poem into a paragraph, but when you load it, it wouldn't look like a poem at all. HTML ignores extra spaces and line breaks between words unless one is specified by a tag like the <br> tag, which creates a line break. To get around this problem, you can use the <pre> tag, which keeps all of its enclosed text in its original formatting. To add Robert Frost's Nothing Gold Can Stay into this webpage, you would have to put it in the "pre" tag using the following code:

      <pre>
        Nature's first green is gold,
        Her hardest hue to hold.
        Her early leaf's a flower;
        But only so an hour.
      </pre>

This code would show up like this: 

          Nature's first green is gold,
          Her hardest hue to hold.
          Her early leaf's a flower;
          But only so an hour.

As opposed to this (same code, with "p" instead of "pre"):

Nature's first green is gold, Her hardest hue to hold. Her early leaf's a flower; But only so an hour.

<Head>

The "head" tag holds all the information that's important for your webpage, but that shouldn't be visible on a webpage. These tags are:

  1. <title>: defines the text that appears on the window tab
  2. <meta>: holds data invisible to page viewers but visible to web crawlers. Includes things like charset (defines the character set, usually "utf-8"), viewport, author, description, etc.
  3. <style>: holds relevant CSS code
  4. <script>: holds relevant JavaScript code that may need to load before the webpage
  5. <link>: links to important files like external JavaScript/CSS

Displaying Tags In A Webpage

To diplay tags in a webpage, you cannot use regular angle brackets, because they will be recognized by your computer as an actual tag and will not appear on your webpage. To combat this issue, replace the "<" symbol with "&lt;" and the ">" symbol with "&gt;". This is known as the "nickname" method of displaying certain charaters that are otherwsise difficult to use when writing HTML code. Other characters that often get written using nicknames instead of a key on a keyboard is the "&", emojis, and characters that use an alphabet other than the Latin one.
