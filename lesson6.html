Lesson 6: File Paths

Absolute Paths

Use the absolute path of a file when linking an external file, as the computer does not have enough information about the file to use a relative path. If you have a file you need to link called project.docx stored in the Documents folder of a computer, the absolute file path of that file may be C:/Users/AI/Documents/project.docx.

It is important to note that the "/" indicates going down a level.

Relative Paths

Relative paths are generally better to use than absolute paths because they are shorter and won't necessarily break every time you reorganize your files, but they can only be used for internal files. If in your HTML webpage project, you have that same "project.docx" file stored the same way, and you were in the AI folder, the relative path of the project.docx file would just be "Documents/project.docx".

To go up a level instead of down a level, one would write "../". When going down a level, you have to specify which file or folder you are using (because any one folder may have more than one child folder), but you don't have to specify so when going up a level (because every file/folder only has one parent folder). So if you were trying to link to a file in the Users folder from the project.docx file, you would write "../../../file2.png" to go up to Documents, then up to AI, then up to Users, and then down to file2.png.
